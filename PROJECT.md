# PROJECT.md

Human-readable documentation for the Ansar Al-Hujjah website project.

## What This Is

A community website for Ansar Al-Hujjah, an Islamic community center in Houston, TX. Visitors see a branded homepage with program cards, and can navigate to dedicated program pages with session notes and resources. There is no backend ŌĆö everything is static HTML hosted on GitHub Pages at ansaralhujjah.org.

The homepage is managed through an admin panel at ansaralhujjah.org/admin, which writes directly to GitHub via the Git Data API. Program sub-pages are static HTML files committed to the repo.

---

## Site Structure

### Homepage (`index.html`)

All CSS, JS, and content in a single file (~1 MB due to embedded base64 flyer images).

Page layout (top to bottom):
1. Logo, title, divider
2. **Admin-managed program cards** ŌĆö injected between `<!-- AAH-ALL-CARDS-START -->` and `<!-- AAH-ALL-CARDS-END -->` markers. All cards created, edited, reordered, or deleted via the admin panel live here.
3. Footer
4. **Floating social icons** ŌĆö fixed-position bottom-right corner. WhatsApp (green) and Instagram (gradient) circular buttons linking to the community chat and Instagram profile. Defined as `.social-float` / `.social-float-btn` CSS in the `<head>` style block and rendered just before `</body>`. These are hardcoded in `index.html`, not admin-managed.

### Program Pages

Each program has its own subdirectory with an `index.html` overview and individual session/article pages:

```
quran-reflections/
  index.html             ŌåÉ program overview (flyer, description, session list)
  notes/                 ŌåÉ Obsidian .md files (source of truth for session content)
    README.md            ŌåÉ workflow instructions
    Session 1 - ....md   ŌåÉ drop Obsidian notes here
  session-1-title/
    index.html           ŌåÉ generated by build_notes.py
  session-2-title/
    index.html           ŌåÉ generated by build_notes.py
```

## Admin Panel

Located at `admin/index.html`. Accessed at ansaralhujjah.org/admin. It is a single-page app that runs entirely in the browser with no server.

**Authentication:** SHA-256 hash of the plaintext password is computed in-browser and compared against the hardcoded `ADMIN_PASSWORD`. Session persists in `sessionStorage` under `aah_authed`.

**On login:** The admin fetches the live `index.html` to sync its local state with whatever is actually published. This prevents the admin from overwriting changes made from another device or browser. See [Sync Behavior](#sync-behavior) below.

**Card management:**
- Create, edit, delete, and reorder event cards
- Edit the two permanent pinned cards (Sisters Social, Quran Reflections) or temporarily remove them from the site
- Upload a flyer image per card (PNG/JPG) ŌĆö gets base64-encoded and embedded in the HTML
- Each card shows a live preview while editing

**Every save, delete, and reorder immediately triggers a publish to GitHub** ŌĆö there is no separate Publish button.

## Card Types

### Pinned Cards
Two permanent cards that always appear in the admin-managed section unless explicitly removed:
- **Sisters Social** (key: `sisters`) ŌĆö links to registration form
- **Quran Reflections** (key: `quran`) ŌĆö links to `/quran-reflections`

These can be edited (title, subtitle, link, icon, flyer) or removed from the site (`removed: true`). They cannot be permanently deleted ŌĆö only hidden. Their state is stored in localStorage under `aah_pinned`.

### Event Cards
Created freely via the admin panel. Identified by base-36 timestamp IDs (e.g., `mo24g5z6`). Stored in localStorage under `aah_events`. Deleted event cards are gone permanently.

### Card Rendering

All admin-managed cards (pinned and event) use the same horizontal `.program-card` format:
- **With flyer** ŌåÆ flyer image displayed as a full-height banner at the top of the card (`height: auto`, no crop), with icon + title + subtitle + arrow in a footer row below.
- **Without flyer** ŌåÆ same layout, but the flyer area shows the card's icon centered on an accent-tinted background (minimum 100px height).

The entire card is a link (`<a class="program-card">`). There is no expand/collapse interaction for admin-managed cards.

## Publishing Flow

The admin panel publishes by writing directly to GitHub using the Git Data API (not the Contents API, which has a 1 MB file size limit that `index.html` exceeds).

Steps on every publish:
1. Get the current branch tip commit SHA
2. Get the tree SHA from that commit
3. Fetch the full recursive tree (`?recursive=1`) to find both `index.html` and `quran-reflections/index.html` blob SHAs
4. Fetch `index.html` blob (full content, no size limit)
5. Replace the content between `<!-- AAH-ALL-CARDS-START -->` and `<!-- AAH-ALL-CARDS-END -->` with freshly-generated HTML for all cards in the current order
6. If the quran card has a flyer, also fetch `quran-reflections/index.html`, replace the content between `<!-- QURAN-FLYER-START -->` and `<!-- QURAN-FLYER-END -->` with the updated flyer image, and include it in the commit
7. Create new blob(s), a new tree, and a new commit via the API
8. Force-update the branch ref to the new commit (`force: true` required because GitHub Pages sometimes causes ref divergence)

Changes are live within ~30 seconds of publish.

## Sync Behavior

Because multiple browsers/devices share the same GitHub-hosted `index.html`, localStorage state can diverge. On every admin login, `syncFromLiveSite()` fetches `../index.html` (the live site) and rebuilds the local state from it:

- Card order is derived from the sequence of `<!-- pinned:key -->` and `<!-- admin:id -->` markers in the HTML
- Event card data (title, sub, link, icon, flyer) is parsed from the HTML around each `<!-- admin:id -->` marker
- Pinned card flyers are parsed from the pinned card HTML
- Any locally-created cards not yet visible in the live HTML are appended to the end of the order

This means local unsaved drafts are lost on login if they were never published.

## Program Pages & Session Notes Workflow

Program sub-pages (e.g., Quran Reflections) are static HTML managed via a Python build script.

### Adding a session note

1. Write your note in Obsidian
2. Copy the `.md` file to `quran-reflections/notes/`
3. Run the build script:
   ```bash
   python3 build_notes.py
   ```
4. Commit and push:
   ```bash
   git push origin main
   ```

### What the build script does (`build_notes.py`)

- Reads all `.md` files from `quran-reflections/notes/` (ignores `README.md`)
- Converts `[[wikilinks]]` to HTML hyperlinks pointing to the corresponding session page URL
- Converts `[[Note Name|Display Text]]` aliased links to the correct URL
- Parses markdown to HTML using the `markdown` package (`extra` + `toc` extensions)
- Wraps output in a styled article page template matching the site design system
- Writes each session to `quran-reflections/{slug}/index.html`
- Injects the session list between `<!-- SESSIONS-START -->` / `<!-- SESSIONS-END -->` markers in `quran-reflections/index.html`

### URL slug rule

Filename (without `.md`) ŌåÆ lowercase ŌåÆ strip special chars ŌåÆ spaces/underscores to hyphens ŌåÆ collapse consecutive hyphens.

| Filename | URL |
|---|---|
| `Session 1 - Introduction.md` | `/quran-reflections/session-1-introduction/` |
| `Session 2 - Ayat 3.md` | `/quran-reflections/session-2-ayat-3/` |

### Dependency

```bash
pip3 install markdown
```

## Constraints and Gotchas

- **`index.html` is ~1 MB.** Uploading large flyer images grows it further. The Git Data API is used specifically because GitHub's Contents API rejects files over 1 MB.
- **The session notes build step is manual.** After dropping `.md` files into `quran-reflections/notes/`, run `python3 build_notes.py` before pushing. The script handles wikilinks, slugs, prev/next nav, and the session list injection automatically.
- **The GitHub token (`GH_TOKEN`) and admin password (`ADMIN_PASSWORD`) are hardcoded** in `admin/index.html`. The token is split into string segments to avoid GitHub's secret scanning triggering on push. The admin panel is `noindex, nofollow` but is not truly private ŌĆö do not store sensitive data in cards.
- **Flyer images in published HTML are permanent until overwritten.** If a card is edited without uploading a new flyer, the existing base64 image is extracted from the live HTML and re-embedded. This is handled by `extractFlyerAt()` and `extractFlyerForPinned()`.
- **WhatsApp and Instagram are floating icons in the bottom-right corner**, not cards. They are hardcoded in `index.html` as `.social-float` and are not admin-managed. Edit directly in `index.html` to change links.
- **Quran Reflections card links to `/quran-reflections`.** The program page hosts the flyer, description, and session list. There is no separate sign-in link on the card.
- **Quran program page flyer syncs automatically on admin publish** ŌĆö when the quran card has a flyer, `publishToGitHub()` also updates the `<!-- QURAN-FLYER-START/END -->` section in `quran-reflections/index.html` in the same commit.
